### Typography System
- Font approach: Geometric Sans
- Size scale:
    - h1: 24px
    - h2: 18px
    - body: 14px
    - labels: 12px
- Weight usage: Bold for primary text, regular for body text, light for labels
- Distinctive moves: None
- Color treatment: Dark text on light background, with primary color used for titles and key information.

### Color Logic
- Mode: light
- Background: #f8f8f8
- Text: 
    - Primary: #000000 (black)
    - Secondary: #808080 (gray)
- Accent: Blue (#0000ff) and Orange (#ffa500)
- The rule: Color is used to highlight key information and calls to action, creating a clear visual hierarchy.

### Layout & Spacing
- Density: Balanced - not too minimal, not too dense
- Grid/spacing unit: 8px base
- Container padding: 16px
- Gap between elements: 8px - 16px
- Chunking strategy: Information is grouped into clearly labeled sections and cards.

### Component Patterns
- **Navigation**: Bottom tabs with custom icons. Icons are simple outlined shapes.
- **Buttons**: Pill-shaped buttons with gradients.
- **Cards**: Rounded containers with minimal content.
- **Data display**: Numbers are displayed prominently with clear labels.
- **Icons**: Custom icons, slightly 3D.

## Use This When
- **Mobile Apps**: Good for casual, user-friendly mobile applications where visual appeal is important.
- **Social Apps**: The playful design works well for social applications targeting a younger audience.
- **Navigation/Maps**: The design would excel for a navigation app.
- **Fun**: When the emotional goal is user delight.

## Avoid This When
- **Enterprise Software**: The playful design is not suitable for serious or professional applications.
- **Data-heavy Dashboards**: The focus on visual appeal over data density makes it unsuitable for dashboards.
- **Accessibility**: The color contrast may need to be adjusted for accessibility.

## Adaptation Recipes

### → Dashboard
- Replace custom icons with standard data visualization icons.
- Reduce the size of the pill shapes.
- Increase data density by using tables and charts.

### → Landing Page
- Use larger, more prominent buttons.
- Add more visual elements, such as images and videos.
- Focus on a clear call to action.

### → Mobile ↔ Web
- **Mobile**: Optimize for touch by increasing touch target sizes to at least 44px.
- **Web**: Use a larger screen layout with more whitespace and horizontal spacing.

### → Invert Color Mode
- Invert the background and text colors.
- Adjust the shadow values to create a sense of depth.
- Keep the accent colors consistent.
